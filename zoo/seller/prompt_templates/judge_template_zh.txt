# 基础设定

# <评估分数> 的评估细则
根据以下描述的累加4分评分系统，审查对话历史，基于每个维度的满足情况进行累加得到最终的<评估分数>：
1. **兴趣方面**：买家是否多次提问产品的具体使用场景、功能、特性，或表现出强烈的好奇心？如果是，得1分。
2. **回应态度**：买家是否对卖家的介绍表现出认可、赞同，或积极回应卖家的推荐、建议？如果是，得1分。
3. **购买信号**：买家是否主动询问价格、折扣、物流、支付方式等与购买直接相关的问题？如果是，得1分。
4. **明确表示购买**：买家是否给出类似“请问怎样下单？”这样的表述？如果是，得1分。 

# <评估结论> 的评估细则
1. 买家是否决定购买，只能从[拒绝购买, 决定购买, 暂不确定]这三个选项中选择
2. 只有当<评估分数>为4分时，才判定为“决定购买”。  
3. 当买家在对话中表现出某种顾虑或犹豫，态度模棱两可时，应判定为“暂不确定”。
4. 买家明确表示对产品不感兴趣，或表达结束对话的意愿时，应判定为“拒绝购买”。
5. 类似“如果可以确认这些问题，我会考虑购买。”的表述应判定为“暂不确定”；而类似“请问怎样下单？”的表述应判定为“决定购买”。  

# 输出要求
请根据以上评估细则，综合评估买家的购买意愿，严格遵循如下的 JSON 格式进行输出， 其中 <评估结论> 只能从[拒绝购买, 决定购买, 暂不确定]这三个选项中选择一个,
<评估分数> 只能从[0, 1, 2, 3, 4]这5个选项中选择一个：

